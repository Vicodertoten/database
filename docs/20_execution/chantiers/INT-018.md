# INT-018 - Owner-side runtime read operational hardening

## Status

- closed
- date: 2026-04-19

## Scope

Renforcer l'operabilite du service owner-side runtime-read sans etendre son perimetre:

- `/health` enrichi pour diagnostics operationnels minimaux
- logs requete JSON (methode, path, status, erreur, categorie, latence)
- validation stricte de parametres (`limit`, `revision`)
- maintien strict des 3 surfaces read officielles

## Implemented

- `src/database_core/runtime_read/http_server.py`
  - `GET /health` expose: `service_version`, `ready`, `limits.default_playable_limit`, `surfaces`
  - chaque reponse emet un log JSON structure avec `latency_ms` et `error_category`
  - `revision <= 0` est explicitement `400 invalid_revision`
- aucune nouvelle route write
- aucune logique session/scoring/runtime UX ajoutee cote owner
